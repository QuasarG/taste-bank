# Mono Console 等宽控制台

## 概述

从一套三栏分析工作台提炼的风格：全站只用一种等宽字体栈，黑白灰纸面配色，
所有容器由 1px 细边框与直角构成，颜色被严格限制在「黑 / 灰 / 粉彩状态签」
三档。层级不靠色彩与字重，而靠字号阶梯（48/24/15/14/12/11/10/9px）与
字距（0.04–0.18em）拉开，信息密度极高但秩序感强。

## 使用场景

- 数据密集的审阅、批处理、监控类工作台与后台工具
- 「列表 + 详情 + 过程/结果」并列的多栏界面
- 流程节点状态面板、维度得分明细、队列进度展示
- 不适合：营销页、品牌官网、需要情绪感染力的 C 端产品

## 设计要点

- **一种字体走全站**：标题、正文、按钮、数字、徽标全部同一等宽字体栈，
  禁止引入无衬线或衬线字体制造「标题感」。
- **标题靠字距不靠字重**：面板标题 13–15px、uppercase、letter-spacing 0.08em，
  字重保持 400–600；品牌字标 letter-spacing 0.18em。
- **细线 + 直角**：所有卡片、面板、对话框都是 1px 边框、0 圆角；
  全站唯一的圆角是 999px 计数徽标，唯一的 50% 圆是加载转环。
- **黑白主交互**：主按钮黑底白字、hover 反转为白底黑字；hover 反馈
  只用浅灰底与边框变深，不用彩色。
- **粉彩只给状态**：蓝/绿/紫/琥珀等粉彩九宫仅用于状态签、等级章、
  警示块，永远是「浅底 + 同色描边 + 深同色系文字」，禁止实心彩色大色块。
- **等级用左侧色条**：条目卡片左侧 4px 实心色条区分等级（紫 S / 蓝 A /
  绿 B / 黄 C），其余边框保持细灰。
- **元信息压小**：辅助文字压到 9–11px，与 12–14px 正文形成密集层级；
  状态章最小可到 8–9px。
- **数字要大**：核心得分用 48px 等宽数字 + 细边框 score band 呈现。
- **动效克制**：150–200ms 的边框色 / 背景 / 位移过渡；运行态用 1px
  外描边脉冲或进度条高光扫过，不用阴影扩散与弹簧曲线。

## Do & Don't

Do:
- 用 1px 细边框切分区域，而不是用阴影与留白硬隔
- 用 summary strip（等分格 + 上数字下标签）展示计数汇总
- 用虚线下划线的行内「引用链」指向证据/出处
- 状态文字用英文大写 chip（READY / RUNNING / DONE）
- 列表卡片给等级左侧色条 + 同色系等级章

Don't:
- 禁止圆角卡片、胶囊按钮（计数徽标除外）
- 禁止实心彩色按钮与大面积彩色背景
- 禁止高饱和渐变（纸面微渐变 #FFFFFF→#F4F6F8 是唯一允许）
- 禁止图标字体与 emoji 图标，状态符号用 ✓ / — / ! 文本字形
- 禁止 900 字重的大标题与居中营销式排版

## 文案语气

按钮与操作词极简，2–4 字：「开始分析」「重新分析」「一键处理」「取消」
「重试」。状态用英文大写单词（QUEUED / RUNNING / DONE）。提示与报错
直接陈述事实，不道歉、不卖萌：「解析完成，生成 6 个段落」「段落 05
统计口径待人工复核」。数字尽量精确给出（14 / 14 项、92%）。

## 文件清单

- `tokens.json` —— 颜色 / 字体 / 字号 / 间距 / 圆角 / 阴影 / 动效变量
- `overrides.css` —— 等级色条、粉彩 chip、计数徽标、节点脉冲、进度扫光、
  引用链、警示块等 tokens 表达不了的 scoped 规则
- `templates/page.html` —— 三栏工作台完整快照（条目库 + 记录详情 + 分析区）
